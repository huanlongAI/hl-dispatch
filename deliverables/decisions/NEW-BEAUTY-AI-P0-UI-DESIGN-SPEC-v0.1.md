## 1. 设计目标

- 让美业老板打开即进入可提问、可产出、可复制的工作界面。
- 降低输入成本，文字和 App 内语音转写并列为 P0 输入。
- 让 AI 输出天然适合人工复核，而不是暗示自动执行。
- 用 `hl-scene-design-system` Flutter adapter 承接 token、组件和状态，避免 App 内自造视觉体系。
- 在轻医美场景中保持经营表达，不诱导专业医疗决策。

## 2. 视觉边界

- 风格：专业、清晰、轻运营工具感，不做娱乐化人格包装。
- 组件：统一使用 `hl-scene-design-system` Flutter adapter 可用组件。
- 色彩：遵循设计系统 token，不新增未登记品牌色。
- 圆角、间距、字号、动效：遵循设计系统 token。
- 图标：使用设计系统或 Flutter 端统一图标方案，避免页面级临时图标资产。
- 页面首屏必须明确产品身份“新美业 AI 老板总助”，但不做营销落地页。
- 禁止用视觉或文案暗示已接入真实经营系统、硬件监听或自动执行。

## 3. 页面清单

| 页面 | P0 内容 | 验收重点 |
|---|---|---|
| 启动/加载 | 品牌名、加载状态、错误重试 | 不出现过度承诺 |
| 登录/授权 | 手机号或账号登录入口，协议确认 | 协议与隐私入口可见 |
| 首次引导 | 门店类型、城市层级、经营目标、当前难题 | 均为手动输入或选择 |
| 首页 | 对话入口、推荐场景、最近草稿、用量入口 | 首屏可直接提问 |
| AI 对话页 | 文本输入、语音转写、场景建议、回答卡片 | 输入和输出状态完整 |
| 模板中心 | 活动、私域、客诉、员工培训、复盘模板 | 模板分类清晰 |
| 草稿详情 | 生成内容、复制、分享、收藏、重新生成 | 明确人工复核 |
| 收藏/历史 | 收藏回答、历史对话、搜索或筛选 | 不展示未授权数据 |
| 用量/订阅 | 剩余次数、积分、套餐说明、升级入口 | 不锁具体支付实现 |

| 合规提示页 | 医疗、隐私、自动执行边界 | 文案明确可理解 |

## 4. 核心交互

### 4.1 对话输入

- 文本输入框常驻底部。
- 语音按钮需要明确按下/录音中/转写中/可编辑状态。
- 语音转写结果进入输入框，用户确认后再发送。
- 输入为空时，展示可点选的经营场景 prompt。

### 4.2 场景化追问

当用户问题过于宽泛时，AI 先追问：
- 门店类型。
- 当前目标。
- 活动周期。
- 预算范围。
- 客群特点。
- 可投入人手。

追问必须是可快速点选和可自由输入并存。

### 4.3 输出呈现

AI 回答优先采用结构化块：
- 结论。
- 操作步骤。
- 风险提醒。
- 可复制文案。
- 复盘指标。

涉及轻医美经营表达时，必须出现边界提醒：仅用于经营、营销、客户沟通、门店管理和服务流程表达。

### 4.4 草稿操作

- 复制：复制当前块或全文。
- 分享：走系统分享，仅分享用户主动选择内容。
- 收藏：保存为收藏项。
- 重新生成：保留原问题，允许补充条件。
- 删除：用户可删除本地可见历史项，服务端删除策略由后续工程确认。

## 5. 状态清单

| 状态 | 页面/组件 | UI 要求 |
|---|---|---|
| 初始空态 | 首页、对话页、历史页 | 给出场景入口，不使用营销空话 |
| 录音中 | 对话输入 | 明确计时、取消、完成 |
| 转写中 | 输入框 | 可见 loading，禁止发送重复请求 |
| 生成中 | AI 回答区 | 展示进度，不遮挡输入 |
| 生成成功 | 回答卡片 | 分块、复制、收藏、重新生成 |
| 生成失败 | 回答卡片 | 可重试，展示简短错误 |
| 用量不足 | 对话页、用量页 | 提示升级或等待刷新，不吞请求 |
| 网络异常 | 全局 | 保留用户输入，支持重试 |
| 合规拒答 | 回答卡片 | 简洁说明边界，并给可回答方向 |
| 未登录 | 受限入口 | 引导登录，不丢失当前草稿 |
| 无权限/账号异常 | 全局 | 明确联系客服或重试路径 |

## 6. 设计师交付物

- Figma 页面流：P0 页面清单全量覆盖。
- 组件映射表：页面组件到 `hl-scene-design-system` Flutter adapter 的映射。
- 状态稿：至少覆盖 §5 状态清单。
- 文案表：按钮、错误、合规提示、空态、用量提示。
- 交互原型：对话输入、语音转写、生成结果、复制分享、用量不足。
- 标注：token 名称、组件名、间距、字号、颜色、圆角、状态。

## 7. AI 协作任务

可交给 AI 辅助生成：
- P0 场景 prompt 初稿。
- 模板中心分类和示例文案。
- 合规拒答与改写建议文案。
- 用户访谈提纲。
- UI 状态验收 checklist。

不得交给 AI 直接定稿：
- 合规边界最终口径。
- 隐私协议和用户协议。
- 支付、退款、订阅法律条款。
- 医疗相关表达最终审核。

## 8. UI 验收标准

- 首屏可直接发起文字提问。
- 语音转写必须由用户主动触发，且有录音中状态。
- 任一 AI 输出都不出现自动执行、后台执行、系统已改动的暗示。
- 设计稿全量使用 `hl-scene-design-system` Flutter adapter 可映射组件或 token。
- 轻医美相关页面不出现医疗诊断、治疗建议、处方、疗效承诺式表达。
- 空态、错误、加载、用量不足、合规拒答均有设计稿。
- 页面文本在小屏宽度下不重叠、不截断关键动作。
- 订阅/积分 UI 只表达边界，不承诺本期支付实现。
