公示边界

允许展示

  • owner 责任域、事项 ID、证据链接状态。
  • 任务派发、接受、进行中、已完成、过程未达标。
  • 团队合盘量化：总派发、接受率、完成量、未达标暴露量。
  • Evidence Pack 完整 / 待补 / 待复核 / 暂缓。
  • L1-L4、qa-verdict、pm-acceptance、release evidence 状态。
  • LTC 正常 / 降级 / 断链已说明 / 系统故障待归因。
  • 事实更正、申诉、补证的处理状态。

禁止展示

  • 绩效分、评级、排名、绩效系数、薪资金额。
  • 态度差、不适配、风险员工等主观或 HR 标签。
  • prompt、模型输出、源文件内容、URL、截图、录屏。
  • 把 token 数、prompt 数、在线时长当作绩效倾向。
  • 把任务量直接等同于绩效好坏。
  • 把 LTC 断链自动转成负面绩效结论。

真实数据接入前置规则

接入真实数据前，必须先完成字段复核与脱敏复核。本页只展示过程事实和补救状态；原始证据、明细链接、模型输入输出、截图录屏、薪酬与绩效结果只进入授权系统，不进入公示页。

数据来源与可展示投影

  • 飞书绩效台账：周期、团队、责任域、任务 ID、派发 / 接受 / 到期 / 关闭状态。
  • GitHub PR / Checks：Issue / PR 编号、门禁结论、证据状态；不展示原始日志。
  • 云效回推：构建 / 部署结论、环境层级、发布证据状态；不展示内部流水线明细。
  • QA Evidence Pack：验收结论、证据包完整度、缺口码、复核状态。
  • LTC aggregate：正常 / 降级 / 断链已说明 / 系统故障待归因；不展示窗口标题、URL、截图或录屏。
  • 授权更正单：更正 ID、处理状态、裁决结论、更新时间；不展示原始申请全文。

脱敏与权限边界

  • 默认展示团队、岗位或责任域编码；实名展示必须有制度依据和授权范围。
  • 证据只展示“完整 / 待补 / 待复核 / 暂缓”等状态，不展示源文件内容。
  • 链接只进入受限证据库；公示页只保留证据是否存在和是否可复核。
  • 员工可见自身明细，团队公开页只看过程状态，管理侧受限页才看完整审计链。
  • 任何人工更正必须保留更正人、时间、依据和前后状态，不允许覆盖原始快照。

员工更正窗口

  • 工作日 18:30 刷新后开放更正；默认 2 个工作日内提交补证、补说明或事实更正。
  • 周期关闭前仍未处理完的更正项，状态必须标为“暂缓”或“待裁决”，不得直接进入结果判断。
  • 更正通过后只更新公示投影，原始快照和更正依据必须保留在审计链中。

刷新与审计留痕

  • 常规刷新节奏为工作日 18:30；紧急刷新必须有授权更正单或管理员发布记录。
  • 每次生成快照必须包含 snapshot_id、source_digest、generated_at 和 operator。
  • 发布后只允许追加更正记录，不允许静默删除历史状态。
