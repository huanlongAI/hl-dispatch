§6 · CODEOWNERS 与所有权边界

以下是站点导览级摘要。具体分支保护、CODEOWNERS 文件和 GitHub 权限以各仓现行配置为准。

hl-contracts

  • RULINGS / governance：Founder。
  • Cap-Spec / HPRD 语义：PM + Founder。
  • OpenAPI / reason code：工程与 PM 共同校验。

代码仓

  • hl-platform：后端工程 Owner。
  • hl-framework：架构 / Infra Owner。
  • hl-console-native：Founder only。

C-L2 场景仓

  • hl-scene-app：前端工程 Owner。
  • hl-scene-design-system：设计 + 前端负责人。
  • 与 C-L1 代码、token、组件硬隔离。

平台附属

  • team-memory：PM 主力，重大治理文档 INV-0 批。
  • sentinel-shared：L4 基建 Owner。
  • tech-cofounder-bot：治理辅助 runtime。

guanghe

  • huanlongAI/guanghe 是 R-070 后权威位置。
  • 唯一维护者：童正辉（INV-0）。
  • 团队不参与建设。

所有权事件

  • create / transfer / rename / archive / delete 必写 OWNERSHIP-LOG。
  • 跨 owner / 跨域 / 可见性变更触发 RULINGS。
  • 未来补登记按 R-070 机制执行。

§7 · 能力包跨仓流动

1

契约冻结

PM 在 hl-contracts 起草能力包，INV-0 合入契约 PR。

2

规格入代码仓

HPRD / 开发设计 / 测试规格落入代码仓 /hprd/，先形成 GitHub 审计基线。

3

Step 5 云效接入

工程师 feature 开发时，云效副本开始并行拉取代码并执行 CI。

4

Status Check 回推

云效 L1-L3 结果回推到 GitHub PR；L4 由 sentinel-shared 提供 advisory。

5

合并与部署

INV-0 按 GitHub Merge，曾正龙在云效执行 staging / canary / prod / rollback。
